Privacy Policy

The short version: everything runs on your machine, and there's no server of mine collecting anything.

This formalizes what's already explained in plain language on the Disclosure page. If the two ever seem to disagree, this page is the accurate one — tell me on GitHub so I can fix the mismatch.
1. Overview

StatusForge, Blipy, and Joystick Companion (collectively "the Software") run entirely on your own machine. There is no backend server operated by this project, no account system, no analytics, and no telemetry. This policy describes exactly what data the Software touches and where it goes — which, for almost everything, is nowhere.

2. What StatusForge reads locally

To detect what game you're playing, StatusForge reads process names, window titles, launch arguments, and — for some emulators — their own log files. These are passive, OS-level reads: the Software never touches a game's or emulator's memory, injects code, or reads save data. This data is used locally to look up the game and, if you've connected a platform, to update its category — it is not stored anywhere beyond your own local game library file, and not sent anywhere except as described below.

3. What leaves your computer

StatusForge makes outbound network calls only in these cases, and only if you've set them up:

  • Twitch and Kick — OAuth-authenticated requests to push a category update, only when you've connected that platform.
  • Steam, GOG, RAWG, IGDB, SteamGridDB, TheGamesDB — metadata lookups (cover art, genre, developer, release year), only when you scan a game.
  • GitHub Releases — a check for newer app versions, which you can disable in Settings.

Streamer.bot automation, if you enable it, is a local websocket connection on your own machine — it never touches the internet.

4. Where your credentials are stored

OAuth tokens for Twitch and Kick are stored in your operating system's own credential store (Windows Credential Manager, macOS Keychain, or the Linux equivalent) — never in plaintext in a config file, and never sent anywhere except directly to that platform's own API.

5. Companion apps

Blipy (optional dual-PC linking) only talks to StatusForge itself, over your own local network — no external accounts, no OAuth, no internet calls of its own.

Joystick Companion (optional Joystick.tv integration) is a separate app with its own OAuth connection to Joystick.tv and its own OS credential-store entry for that token, independent of StatusForge's. It polls StatusForge's local status endpoint (using a token you generate) to know what game is currently detected — that request stays on your machine.

6. Third-party privacy policies

Once data reaches a third-party platform (Twitch, Kick, Joystick.tv, Steam, GOG, RAWG, IGDB, SteamGridDB, TheGamesDB, GitHub), that platform's own privacy policy governs it — this project has no visibility or control over how they handle it.

7. This website

This site is static and hosted on GitHub Pages. It doesn't set its own cookies or run analytics. GitHub itself may log standard server access data (IP address, request time) as part of hosting the site — see GitHub's Privacy Statement for that.

8. Children's privacy

The Software isn't directed at children under 13, and doesn't knowingly collect any data from anyone — children included, since it doesn't collect data from anyone at all.

9. Changes to this policy

This policy may change as the Software changes. Material changes will update the date below.
